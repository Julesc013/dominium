# Dominium AIDE Preservation Contract

Status: active for future AIDE-controlled work

Future AIDE tasks in Dominium must preserve:

- `AGENTS.md` manual doctrine and managed-section boundaries.
- `.aide/memory/**`.
- `.aide/queue/**`.
- `.aide/evals/**`, including Dominium-specific golden tasks.
- `.aide/context/dominium-doctrine-refs.md`.
- `.aide/context/latest-*` unless regenerated by Dominium-local AIDE for the active task.
- Dominium doctrine, canon, specs, data, contracts, and governance references by path, not by pasted doctrine dumps.
- XStack, AuditX, RepoX, TestX, BuildX-like systems, and existing validators until wrapped or migrated with evidence.
- Product source roots: `runtime/**`, `engine/**`, `game/**`, `apps/**`, `content/**`.
- Existing build/test/release validators and command matrices.
- Local ignored state, especially `.aide.local/**`, `.dominium.local/**`, build/out/dist directories, raw prompts, raw responses, provider state, and secrets.
- Generated outputs unless explicitly regenerated by target-local AIDE in the current task.

Future AIDE tasks must not:

- Delete, rename, move, or retire legacy tool systems without wrapper/migration evidence and explicit reviewed scope.
- Move roots, create aliases, create shims, or rewrite references without reviewed maps.
- Treat generated outputs as authority unless a stronger repo artifact promotes them.
- Call providers/models/network unless a future reviewed task explicitly enables live execution.
- Mutate branches, tags, GitHub settings, CI, releases, or protected doctrine without explicit scope and review.
