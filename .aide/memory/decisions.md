# Dominium AIDE Decisions

- Import only portable AIDE Lite files needed for target-local context,
  validation, packet generation, and evidence.
- Do not copy AIDE source queue history or source project memory.
- Do not copy AIDE generated context, generated reports, latest route decisions,
  cache-key reports, Gateway status reports, provider status reports, local
  traces, raw prompts, raw responses, `.aide.local/`, `.env`, or secrets.
- Generate Dominium snapshot, context, task packet, review packet, token
  estimates, and evidence inside Dominium.
- Keep Dominium doctrine in repo docs and specs as path-referenced authority,
  not as AIDE memory dumps.
- Preserve existing Dominium `AGENTS.md` content; add only a managed compact
  AIDE guidance section.
- Make no Dominium product feature, runtime, engine, client, server, simulation,
  or model changes during Q23.
