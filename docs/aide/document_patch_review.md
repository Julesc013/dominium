# AIDE And Codex Document Patch Review

AIDE and Codex patches are not hidden mutation when they enter product or Workbench law. They must be reviewable transactions with intent, command linkage, capability/refusal posture, patch envelope, validation/dry-run result, diagnostics, evidence, and commit/refusal state.

For repository work, agents may still edit files as part of a task. When the edited surface is authority-bearing or future Workbench-visible, the task must also leave evidence that maps the change to document/patch/transaction law or explicitly state why the surface remains contract-only.

CLI/headless scripts use the same envelopes. A scripted change to pack, mod, theme, module descriptor, schema, command, release package, or future save/world mutation proof should produce the same fields that a Workbench transaction would produce.

A review packet should include changed document refs, patch IDs, transaction IDs, dry-run status, validators run, diagnostic/refusal refs, evidence packet refs, rollback policy, non-goals, and remaining implementation gaps.
