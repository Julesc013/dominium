| Generated outputs | tracked generated C/H/JSON files under `libs/appcore/ui_bind/` |
| Committed? | yes, as generated source artifacts consumed by `libs/appcore` |

The failure was not stale semantic content after regeneration. It was byte-for-byte freshness drift caused by Windows checkout line-ending normalization: `core.autocrlf=true` and no attribute pinned `libs/appcore/ui_bind/**` to LF. `tool_ui_bind` compares bytes exactly, so CRLF-normalized generated files failed freshness even when their normalized Git content matched the generated content.
