## Routing Decisions

- `apps/workbench/module/game/edit/` was a user-facing Workbench game editor module, so it moved to `apps/workbench/module/game/editor/`.
- `apps/workbench/module/tool/editor/` was a user-facing editor for tool/UI documents, so it moved to `apps/workbench/module/tooling/editor/`.
- `apps/workbench/module/tool/editor/tool_editor_docgen.cpp` was noninteractive UI document generation, so it moved to `tools/codegen/ui/tool_editor/tool_editor_docgen.cpp`.
- `apps/workbench/module/ui/editor/gen/` contained checked-in generated action bindings consumed by the Workbench UI editor, so it stayed with the module but was renamed to `apps/workbench/module/ui/editor/generated/`.
- `apps/workbench/module/ui/native/` was a Workbench-specific native preview/editor host, not reusable native UI substrate, so it moved to `apps/workbench/module/ui/preview/native/`.
- `apps/workbench/module/ui/preview/support/` held module-local preview loading and validation helpers, so it moved to `apps/workbench/module/ui/preview/service/`.
