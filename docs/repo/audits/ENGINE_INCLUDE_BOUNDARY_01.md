## Route Decisions

- Runtime-facing app, CLI/TUI, GUI/UI, input, IO/storage, package, and render public headers must live under runtime-owned include paths if reintroduced.
- Audio runtime API belongs under `runtime/include/domino/audio.h`.
- Canvas/render command recording API belongs under `runtime/include/domino/canvas.h`.
- Package/mod registry and launcher-extension runtime API belongs under `runtime/include/domino/mod.h`.
- Game/world/domain public headers must live under game-owned include paths if reintroduced.
- `engine/include/` remains reserved for deterministic substrate APIs. Broader review of flat legacy-style headers such as `engine/include/domino/dworld.h`, `engine/include/domino/dnet.h`, `engine/include/domino/authority.h`, and domain-prefixed `d*` headers remains follow-up work because those headers need a separate semantic classification/API strategy pass.
