## Route Decisions

- Runtime-facing app, CLI/TUI, GUI/UI, input, IO/storage, package, and render public headers must live under runtime-owned include paths if reintroduced.
- Game/world/domain public headers must live under game-owned include paths if reintroduced.
- `engine/include/` remains reserved for deterministic substrate APIs. Broader review of flat legacy-style headers such as `engine/include/domino/dworld.h`, `engine/include/domino/dnet.h`, or domain-prefixed `d*` headers is explicitly follow-up work because the task's retired-path inventory did not contain the named directory leaks and those flat headers need a separate semantic classification pass.

## Include Namespace Policy

- `domino/` under `engine/include/` is for shared deterministic substrate APIs.
- Runtime host, shell, UI, render, input, package, platform, and storage APIs belong under runtime-owned include paths.
- Dominium game/world/domain meaning belongs under game-owned include paths, with `dominium/` preferred for product/game-facing public API where the existing game include tree uses that namespace.
