## 3. The Core Simulation Philosophy

The simulation philosophy is that truth is lawful, perception is filtered, rendering is presentation, and mutation must pass through deterministic process.

**Why this chapter matters.** Canon and architecture docs make truth/perceived/render separation and process-only mutation the floor. The corpus repeatedly applies that floor to Workbench, UI, providers, content packs, and future world simulation.

> [!CURRENT_TRUTH] Current repo truth comes first in this chapter. Archive and conversation evidence is used to explain design intent, recurring concerns, and review candidates without promoting it.
