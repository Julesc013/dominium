# In-Chat Reader — Dominium Omega/Xi/Pi Architecture & Future-Proofing Planning

## Package overview

This preservation package captures the substance of the chat: what was discussed, why it mattered, what was decided, what remains pending, what is uncertain, and how a future chat should continue.

## File index

- `00_manifest`: package index.
- `01_human_readable_report`: full human-readable report.
- `02_context_transfer_packet`: future-chat handoff.
- `03_spec_sheet`: machine-friendly YAML spec.
- `04_registers`: workstream/decision/task/etc. registers.
- `05_aggregator_packet`: compact cross-chat merge packet.
- `06_reader_brief`: short readable brief.
- `07_verification_and_audit`: self-audit and verification queue.
- `08_future_chat_bootstrap_prompt`: prompt for a new chat.
- `09_in_chat_reader`: this guide.

## Best reading order

1. Reader brief.
2. Human-readable report sections 0–16.
3. Context transfer packet.
4. Registers.
5. Spec sheet and aggregator packet if building a master spec book.
