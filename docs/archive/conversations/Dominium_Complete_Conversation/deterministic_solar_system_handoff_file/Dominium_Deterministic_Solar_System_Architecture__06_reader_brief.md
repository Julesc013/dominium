# Reader Brief — Dominium Deterministic Solar-System Architecture

This chat defined a major direction for Dominium: a deterministic robotic seed-civilisation game where players are manufactured robot bodies spawned by a mothership, then expand across scientifically generated planets by surveying, mining, building power, refining materials, fabricating machines, constructing spawn labs, automating cities, terraforming, and eventually building megaprojects.

## Top 20 things to know

1. Domino should probably be the deterministic simulation/data core.
2. Unreal should not be assumed to own the authoritative universe.
3. Planets should be science-bounded procedural outputs, not arbitrary slider toys.
4. Designer painting should be an override layer, not the main workflow.
5. Save files, packs, painted fields, terraforming, and construction should share field/delta technology.
6. The main lore premise is a robotic mothership sent by humans.
7. Players spawn as robot bodies from physical machines.
8. Remote spawn labs are a key expansion milestone.
9. The mothership provides recipes/knowledge but has finite resources and low throughput.
10. Robot bodies explain the HUD and strong inventory/carry capacity.
11. Nanobots are the construction interface, but must consume materials, energy, time, and heat budget.
12. Worker NPC labour was strongly deprioritised.
13. Automation and process graphs are central.
14. Machines/factories should compile into aggregate equations when distant.
15. Cities should be easy to start but advanced industry should require large supply chains.
16. Fog of war is a data/security model, not merely visual shading.
17. Client-shared compute must not expose hidden MMO truth.
18. Millions across shards/universe is different from millions in one real-time area.
19. The proposed first vertical slice is mothership-to-remote-spawn-lab.
20. Many assistant suggestions are not yet accepted decisions.
