## Ultra-Condensed Carry-Forward Capsule

This chat is a major architecture and planning session for Dominium/Domino. It began with a concrete launcher UI problem and a plan for a Windows-only UI Editor, expanded into Tool Editor and UI automation prompts, then pivoted away from those as final products. The accepted direction became Dominium Workbench: a rendered, modular, command-driven, agent-aware production environment over shared Dominium/DOMino contracts, services, providers, packs, modules, documents, patches, projections, diagnostics, and evidence. Workbench is not authority; it is the richest surface over the system.

The chat established a unified presentation doctrine: CLI, TUI, rendered GUI, OS-native GUI, headless reports, Workbench panels, tests, and AIDE/Codex workflows should all project the same typed command/result/refusal/document/evidence truth. TUI is first-class. Rendered GUI is cross-platform and data-described. OS-native GUI is optional projection, not semantic authority. Themes and OEM+ OS-style mimicry are possible through token/control-skin packs and primitive rendering, with richer assets optional.

The chat also established a reusable vocabulary: component, service, provider, pack, module, workspace, app, artifact. It developed engineering-law guidance for modularity, portability, ABI, schema/versioning, generated artifacts, dependency direction, conformance tests, and compatibility.

AIDE/Codex workflow was a major focus. AIDE is scheduler/ledger/gatekeeper; Codex is bounded patch executor. Development can be non-blocking; promotion is evidence-blocked. Many Codex prompts were generated for queue reconciliation, replay proof, barebones client, AIDE workflow law, WorkUnit schema, dev/main policy, checkpoint loop, capability reality ledger, and full-gate legacy test routing.

Provider strategy was refined: use raylib/rlgl/rlsw/raygui/raudio/SDL2/Lua aggressively as seed providers, but keep service identity first-party and fence third-party types from engine/game/contracts/content/saves/replays/public SDK.
