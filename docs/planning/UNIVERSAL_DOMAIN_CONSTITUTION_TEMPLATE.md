# Universal Domain Constitution Template

## A. Purpose And Scope

This artifact is `POST-ZETA-2`.

It freezes the reusable drafting template every future domain constitution must satisfy.

It extends `SPEC_DOMAIN_CONTRACT_TEMPLATE`.
It does not replace it.

Its goal is to prevent future domain constitutions from becoming:

- ad hoc essays
- implementation sketches without semantic structure
- domain-specific ontologies that forget the universal substrate

## B. Mandatory Template Sections

Every domain constitution must contain all of the following sections.

### B.1 Identity And Scope

Must declare:

- `domain_id`
- title and summary
- governed phenomena
- explicit non-goals
- canonical owning roots and overlap zones

### B.2 Primitive Substrate

Must declare:

- primitive substrate units or truth-bearing commitments
- conserved or bounded quantities
- identity anchors
- substrate invariants

### B.3 Processes

Must declare:

- lawful Process mutation paths
- inputs and outputs
- refusal conditions
- degradation conditions

### B.4 Recognitions

Must declare:

- recognized structures over substrate
- recognition validity conditions
- observer-dependence or observer-independence

### B.5 Affordances And Capability Hooks

Must declare:

- domain affordance families
- actor classes served
- capability-surface hooks or delegated references
- authority or legality constraints

### B.6 Formalizations

Must declare:

- declared formalization stages already relevant
- blueprints, protocols, standards, or institutional forms if any
- `none_declared` explicitly when absent

### B.7 Representations, Substitutions, And Capsules

Must declare:

- lawful representations
- substitution or capsule posture
- macro and micro expectations
- expansion and refusal triggers

### B.8 Failures And Refusals

Must declare:

- ordinary failure classes
- degraded states
- unsafe or invalid states
- refusal semantics

### B.9 Verification

Must declare:

- invariants to verify
- evidence surfaces
- replay, determinism, or equivalence expectations where relevant
- audit or review obligations

### B.10 Observation Bindings

Must declare:

- lawful observation channels
- epistemic limitations
- summary or redaction rules

### B.11 Interface Bindings

Must declare posture for:

- embodied players
- abstract planners
- AI agents
- institutions
- automation
- future interfaces

### B.12 Player Desires Served

Must declare:

- which player or civilization desire families the domain serves
- what the domain refuses or leaves out
- what acceptance evidence later drafting must satisfy

### B.13 Civilization Scaling Path

Must declare:

- micro to macro shape
- continuity obligations across scale
- what higher-scale summaries remain lawful

### B.14 Mod And Extension Path

Must declare:

- extension points
- protected invariants
- compatibility and refusal rules for extensions

### B.15 Stability And Compatibility Metadata

Must declare:

- contract version
- stability class
- compatibility notes
- replacement target if provisional
- downstream bridge dependencies

## C. Required Drafting Posture

Every domain constitution must also satisfy these drafting rules:

- inherit `Λ-0` reality law directly
- use the cross-domain bridge template instead of inventing custom bridge grammar
- use the player-desire matrix for acceptance reasoning instead of vague aspiration
- distinguish doctrine, admission, and realization explicitly when frontier topics appear
- record overlap and ownership sensitivities rather than hiding them

## D. Minimum Completeness Test

A domain constitution is not draft-ready unless all of the following are explicit:

- identity and scope
- primitive substrate
- Process model
- recognition model
- failure and refusal model
- verification posture
- observation versus truth distinction
- scale path
- extension path
- compatibility and stability metadata

## E. Forbidden Domain Drafting Shapes

The following are forbidden:

- local redefinition of truth, materialization, or continuity
- observation or UI being used as ontology
- implicit substitution rules
- missing failure semantics
- missing verification posture
- domain scope inferred from directory names alone
- player-desire claims with no acceptance evidence expectations

## F. Template Outcome

The reusable domain constitution floor is now frozen.

Future domain constitutions may specialize this template, but they may not omit its sections or collapse its distinctions.
