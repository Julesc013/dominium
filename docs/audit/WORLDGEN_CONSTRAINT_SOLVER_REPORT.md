## Supported Constraint Operators

The deterministic solver currently supports:

1. `eq`
2. `range`
3. `min`
4. `max`
5. `set_membership`
6. `custom_tag`

Supported scoring operators:

1. `maximize`
2. `minimize`
3. `target`

Tie-break policies:

1. `lexicographic`
2. `seed_order`
3. `explicit_field`

## Deterministic Search Summary

Deterministic candidate generation is data-driven by `deterministic_seed_policy`:

1. `single`: exactly one candidate, seed = `base_seed`
2. `multi`: candidate list generated by deterministic hash expansion `H("rng.worldgen.constraints.search|base_seed|i")`

Selection ordering is deterministic:

1. hard constraints evaluated first
2. soft/scoring totals computed in fixed iteration order
3. candidates ranked by `(soft_score desc, hard_pass_count desc, tie_break_policy)`
4. `selected_seed` = first ranked candidate

The emitted `worldgen_search_plan` includes deterministic hash material and is validated against canonical schema.

## Refusal Semantics

Implemented refusal paths include:

1. `refusal.constraints_unsatisfiable`
2. `refusal.search_exhausted`
3. `refusal.seed_policy_invalid`
4. `REFUSE_WORLDGEN_CONSTRAINTS_NOT_REGISTERED` (selected constraints missing from compiled registry)

No hidden fallback behavior is used.

## Known Limitations

1. Constraint metrics are currently preview metrics, not full world artifact simulation metrics.
2. `custom_tag` is a structural placeholder operator with deterministic semantics but limited expressiveness.
3. Multi-seed search is deterministic but not yet optimized for large candidate counts.

## Extension Path

Future expansion remains pack/registry-driven:

1. add constraints via `worldgen_constraints` pack contribution type
2. register IDs in `data/registries/worldgen_constraints_registry.json`
3. validate schemas through CompatX
4. add domain-specific metrics without introducing new primitives

## Primitive and Semantic Safety

No new simulation primitives were introduced.  
No solver physics/math semantics were changed outside deterministic structural search selection.

## Domain/Contract Interaction

Worldgen constraints remain structural and compatible with Domain/Contract governance:

1. constraints select deterministic seeds and plans
2. resulting artifacts still flow through existing domain/contract registries
