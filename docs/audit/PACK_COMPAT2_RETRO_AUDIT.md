# PACK-COMPAT-2 Retro Audit

## Scope
Audit existing persistent artifact surfaces before introducing deterministic format versioning and migration hooks.

## Findings

### Save Files
- Authoritative save state is currently persisted as `universe_state.json`.
- Current load/write paths:
  - `tools/xstack/sessionx/creator.py`
  - `tools/xstack/sessionx/runner.py`
  - `tools/xstack/sessionx/script_runner.py`
  - `src/server/server_boot.py`
  - `src/server/server_console.py`
- `schemas/universe_state.schema.json` does not currently describe:
  - `format_version`
  - `semantic_contract_bundle_hash`
  - `engine_version_created`
  - top-level `deterministic_fingerprint`

### Blueprint Files
- Blueprint payloads are loaded directly from disk in `src/materials/blueprint_engine.py`.
- Existing `schemas/blueprint.schema.json` does not currently describe:
  - `format_version`
  - `engine_version_created`
  - `required_contract_ranges`
  - top-level `deterministic_fingerprint`
- Existing pack blueprint payloads must remain usable without in-place edits.

### Profile Bundles / Session Templates
- `profiles/bundles/bundle.mvp_default.json` and `data/session_templates/session.mvp_default.json` are generated by `tools/mvp/runtime_bundle.py`.
- There is no dedicated JSON schema for:
  - profile bundle
  - session template
- Current payloads already carry deterministic hashes, but not explicit format metadata.

### Pack Locks
- `schemas/pack_lock.schema.json` currently lacks:
  - `format_version`
  - `engine_version_created`
- Pack lock generation occurs in:
  - `tools/mvp/runtime_bundle.py`
  - `src/packs/compat/pack_verification_pipeline.py`

### Migration Tooling
- Generic CompatX migration helpers already exist in `tools/compatx/core/migration_runner.py`.
- Existing migration registry `data/registries/migrations.json` is governance-oriented and not the artifact-level registry requested for PACK-COMPAT-2.
- No shared artifact loader currently:
  - detects older/newer formats
  - applies deterministic migration chains
  - reports read-only fallback status

## Integration Points Required
- Shared format loader/stamper for save, blueprint, profile bundle, session template, and pack lock artifacts.
- Deterministic migration registry keyed by artifact category and format version.
- Loader integration into:
  - save/session runners
  - server boot/snapshot
  - blueprint loading
  - runtime bundle reads
  - pack-lock generation and validation

## Non-Goals Confirmed
- No simulation, worldgen, or logic semantics need to change.
- No artifact contents in packs need to be rewritten in place.
- No online services are required.
