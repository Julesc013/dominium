# Content Profiles

`content/profiles/` contains authored runtime, user, and game profile payloads.

Provider/build/release recipes belong under `release/profiles/`. Contract law
and schemas for profiles belong under `contracts/profile/`.

Do not recreate a top-level `profiles/` root.
